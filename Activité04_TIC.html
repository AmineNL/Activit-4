Les techniques de l’information et de la communication (TIC)

Les technologies de l'information et de la communication regroupent un ensemble de ressources techniques (matériel et logiciel) nécessaires à la mise en œuvre des services de l'information et de la communication pour produire, manipuler, convertir, stocker, gérer, transmettre et retrouver l'information et pour communiquer. En utilisant les techniques de L’informatique et Internet, l’audiovisuel, Multimédia et des télécommunications.

Catégories et objectifs

Les TIC sont présentés dans 3 grandes catégories
La téléinformatique qui s’occupe du traitement de l’information sur un ordinateur distant. Télécommunication : toute transmission, émission et réception à distance, de signes, de signaux, d’écrits, d’images, de sons ou de renseignements de toutes natures. L’électronique, les composants, l'équipement grand public.

Composants de TIC

De manière concrète, elle se compose de deux éléments, les logiciels, le matériel informatique, les réseaux et toute autre ressource TIC tangible.

Objectifs recherchés

Le but est d’assurer l'extensibilité, la connectivité, la compatibilité et la fiabilité de l'infra- structure TIC

Caractéristiques

La rapidité dans la réalisation des activités. La décentralisation du pouvoir Un meilleur accès aux informations La rationalité donner un avantage pour l'entreprise et la mondialisation La répartition, transmettre et recevoir des informations dans n’importe quel temps et de n’importe quelle région.

Initiation aux techniques de communication

On distingue trois principaux types d'objectifs de communication : Les objectifs cognitifs faire connaître, faire identifier Les objectifs affectifs faire adhérer, convaincre : Les objectifs comportementaux, agir sur les pratiques.

Réseau ( informatique ) Network

Un réseau informatique est un ensemble d’ordinateurs, de terminaux et de périphériques connectés entre eux pour échanger des informations numériques selon des règles bien définies. Il existe deux notions fondamentales sur lesquelles s’appuie un réseau : l’interconnexion et la communication.

Les caractèristiques de base d’un réseau

Les caractéristiques de base d’un réseau incluent la topologie, qui définit l’architecture du réseau, le Débit, qui mesure la quantité de données numériques transmises par Seconde, la distance maximale que les données peuvent parcourir et le nombre maximum de nœuds pouvant être interconnectés

Quelques terminologies en réseau

Ce titre présente également des termes couramment utilisés en Réseau, tels que la station de travail, le nœud, le serveur et le Paquet. Il est expliqué que les réseaux peuvent être homogènes, C’est-à-dire que tous les ordinateurs appartiennent au même Constructeur, ou hétérogènes, où les ordinateurs appartiennent à des Constructeurs différents.

Éléments d’un réseau

Ce titre énumère les éléments nécessaires pour établir un Réseau, y compris les ordinateurs équipés d’une carte de Communication, les logiciels tels que les navigateurs et les clients de messagerie, les supports tels que les câbles Ethernet et les équipements d’interconnexion tels que les commutateurs et les routeurs.

Le Modèle OSI (Open Systems Interconnection)

Il est un modèle de référence utilisé pour la conception de Réseaux informatiques , qui divise les communications en sept couches : La couche physique , La couche liaison de données ,La couche réseau, La couche transport , La couche session , La couche présentation , La couche application .

Le Modèle TCP/IP (Transmission Control Protocol/Internet Protocol)

Il est utilisé pour la conception et la mise en Œuvre des réseaux informatiques modernes . Il Divise les communications en Quatre couches principales : La couche réseau , La couche liaison de données , La couche application , La couche transport .

L’adressage

Il y’a deux mode d’adressage IPV4 et IPV6 : Adresse source sur (32 bits) et (128 bits) .

Introduction

Le web et internet c’est pareil Le WWW (WORLD WIDE WEB), littéralement la « toile mondiale ». Souvent les gens confondent entre le web et internet sachant que le Web n’est qu’une partition (application) d’Internet, de plus Le Web a été inventé par Tim Berners-Lee et Robert Cailliau plusieurs années après Internet. Son principe est d’accéder à des documents textuels situés sur des machines accessibles par Internet. Internet : support de communication général. Web : une partie des contenus circulant sur l’Internet.

Aspects techniques du Web

Les 3 mécanismes de base du Web sont : URL (Le Web permet d’accéder à un ensemble de ressources, le mécanisme de localisation peut faire appel au protocole DNS). HTTP (Protocole de niveau applicatif, paradigme client-serveur, protocole sans état). HTML (Langage de description de « pages Web » (texte, images…), programmation déclarative).

Page web et site web

Une page Web, c’est un document multimédia : images, textes, liens ...etc. Décrit élément par élément : titre, morceaux de texte, images...etc. • Avec un langage de description :HTML (1992), XHTML (2002) ou HTML5 et stocké dans un fichier : html (où .htm, .xhtml). • Une page Web est interprétée et affichée par un navigateur, elle est localisée sur Internet à l’aide d’une adresse (URL) et qui permet d’accéder à d’autres pages en suivant des liens. • Un site web est un regroupement de pages Web autour d’une thématique commune, reliées entre elles par des liens hypertextes.

Création d’une page web HTML

Un document HTML (HyperText Markup Language) est : un fichier texte construit suivant une syntaxe précise, structuré par des balises et interprété dans un navigateur (Mozilla Firefox, Safari, etc.) puis est créé dans un éditeur de pages web (Dreamweaver......) ou tout éditeur de texte (bloc, notes, wordpad......). Les principes de langage HTML : Les balises : doit comporter au minimum ces 4 balises : < html> ... < /html> : Délimite le début et la fin du document. < head> ... : Entête du document. < title> ... < /title> : Titre du document. < body> ... < /body> : Corps du document. Les titres et paragraphes : ... : Titre de niveau n :1 à 6. Exemple < h1> Grand titre < /h1>. < p> ... < p> : paragraphe.

    • Introduction

    Ce chapitre introductif traite de la définition de Git et GitHub pour éviter toute confusion, ainsi que de l'explication de la popularité de Git et pourquoi les systèmes de gestion de version sont devenus indispensables aujourd'hui.

    Qu’est-ce que Git ?

    Git est un logiciel de gestion de version créé en 2005 par Linus Torvalds, le créateur de Linux. Il permet de conserver un historique des modifications effectuées sur un projet, afin de pouvoir rapidement identifier les changements et de revenir à une ancienne version en cas de besoin. Les logiciels de gestion de version sont devenus indispensables pour la gestion de projets et le travail d'équipe efficace. Git est le leader incontesté des logiciels de gestion de version et il est donc essentiel pour tout développeur de savoir l'utiliser.

    A quoi sert concrètement un système de gestion de version ?

    Un système de gestion de version permet de conserver l'historique des modifications effectuées sur un projet, ce qui est particulièrement utile lorsque plusieurs personnes travaillent sur un même projet. Cela permet de coordonner le travail entre les différents développeurs, d'éviter les conflits entre les différentes versions des fichiers et de restaurer une version précédente en cas de problème. Git permet de synchroniser les versions locales avec celles du serveur, de se connecter à d'autres projets et de mettre à jour automatiquement les différences. Les développeurs ont accès à l'historique des modifications pour l'intégralité du projet, ce qui leur permet de savoir quels changements ont été effectués et par qui.

    Les deux modèles des logiciels de gestion de version : modèle centralisé vs modèle décentralisé

    Les logiciels de gestion de version peuvent être basés sur deux modèles : le modèle centralisé et le modèle décentralisé (ou distribué). Dans un modèle centralisé, la source du code est hébergée sur un serveur central et les utilisateurs se connectent à ce serveur pour travailler. Dans un modèle décentralisé, chaque utilisateur télécharge et héberge l'intégralité du code source du projet sur sa propre machine. Git est un exemple de modèle décentralisé, qui offre des avantages tels que la simplicité et la flexibilité du travail ainsi que la sécurité grâce à la présence de copies du projet sur les machines des utilisateurs.

    Qu’est-ce que GitHub ?

    Le logiciel Git permet d'enregistrer les différentes modifications effectuées sur un projet et de revenir à une version précédente du projet. Un dépôt (ou "repository" ou "repo") est la copie de l'intégralité des fichiers d'un projet et de leur version située sur le serveur central. GitHub est un service en ligne qui héberge des dépôts Git et est le plus grand hébergeur de dépôts Git du monde. Les dépôts hébergés sur GitHub sont majoritairement publics, ce qui signifie que n'importe qui peut télécharger le code et contribuer à leur développement en proposant de nouvelles fonctionnalités.

    Utiliser Git : ligne de commande, console et interface graphique

    Il existe différents types d'interfaces pour utiliser Git, notamment la ligne de commande et l'interface graphique. Dans ce cours, la console en ligne de commande est préférée car elle donne accès à toutes les commandes Git et permet une cohérence pour tous les utilisateurs. De plus, si vous connaissez la ligne de commande, vous pouvez facilement utiliser n'importe quelle autre interface. Pour ouvrir la ligne de commande sur Windows, vous pouvez utiliser le raccourci clavier ou rechercher "cmd" dans la barre de recherche.

    Installation de Git

    Le processus d'installation de Git peut-être effectué en téléchargeant la dernière version depuis le site officiel http://git-scm.com/downloads et en suivant les instructions à l'écran en laissant les valeurs par défaut.

    Paramétrage de Git

    Le paramétrage de Git consiste à renseigner un nom d'utilisateur et une adresse mail pour enregistrer certaines données. Pour cela, on utilise la commande "git config" qui permet de modifier les variables de configuration de Git. On ajoute également l'option "--global" pour indiquer que ces informations doivent être utilisées globalement. On peut vérifier que les informations ont bien été enregistrées en utilisant les commandes "git config user.name" et "git config user.email".

    Démarrer un dépôt Git

    Pour démarrer un dépôt Git, il faut importer ou cloner un projet. Il y a deux façons de créer un dépôt : importer un répertoire existant dans Git ou cloner un dépôt Git existant. Avant cela, il est important de comprendre le fonctionnement général de Git et comment il gère les informations.

    La gestion des informations selon Git

    Les données sont considérées comme des instantanés (snapshots) de l'état du projet à un moment précis, contrairement aux autres systèmes de gestion de version qui stockent des différences entre les versions. Chaque instantané est stocké dans une base de données locale et Git enregistre une référence à cet instantané pour y accéder ultérieurement. Cette approche permet de travailler localement sur l'historique complet du projet sans avoir à être connecté à un serveur central distant, ce qui accélère les opérations et rend le travail plus agréable.

    Les états des fichiers

    Dans Git, un fichier peut être suivi ou non suivi. Un fichier est considéré comme suivi s'il appartient au dernier instantané enregistré dans Git. Lorsqu'on démarre un dépôt Git, les fichiers sont au départ tous non suivi et doivent être indexés et validés pour être enregistrés dans la base de données locale. Un fichier suivi peut être dans l'un des trois états suivants : modifié, indexé ou validé. Un fichier modifié est considéré comme modifié tant qu'il n'a pas été indexé. On indexe un fichier pour indiquer à Git qu'il doit être inclus dans le prochain instantané. Lorsqu'on demande à Git de prendre un instantané, les fichiers indexés et non modifiés font partie de l'instantané et sont à nouveau considérés comme validés.

    Les zones de travail

    Le système de gestion de versions Git utilise trois zones de travail pour gérer les états des fichiers : le répertoire de travail (working tree), la zone d'index (staging area) et le répertoire Git (repository). Le répertoire de travail correspond à l'extraction d'une version du projet où les fichiers sont extraits de la base de données compressée et placés sur le disque pour être utilisés ou modifiés. La zone d'index est un fichier qui stocke les informations sur ce qui fera partie du prochain instantané ou du prochain "commit". Le répertoire Git est l'endroit où les méta-données et la base de données des objets de votre projet sont stockées. Le processus de travail consiste à travailler sur les fichiers dans le répertoire de travail, indexer les fichiers modifiés ou créés dans la zone d'index, et valider la version indexée des fichiers pour les ajouter au répertoire Git.









    Créer un dépôt Git à partir d’un répertoire existant

    Lorsqu'on démarre avec Git et qu'on a déjà des projets en cours, on peut les importer dans Git en créant un dépôt Git à partir d'un répertoire existant. Cela peut être fait en utilisant la commande "git init" pour initialiser le dépôt Git.
    La commande "git status" permet de vérifier l'état des fichiers dans le répertoire. Elle affiche les fichiers qui ne sont pas encore sous suivi de version ("untracked") et informe sur la branche actuelle.
    Pour indexer des fichiers dans Git, on utilise la commande "git add". On peut spécifier le nom d'un fichier, le nom d'un répertoire pour indexer tous les fichiers qu'il contient, ou utiliser des schémas de correspondance avec les fileglobs.
    Après avoir indexé les fichiers, on peut les valider en utilisant la commande "git commit". Il est recommandé d'ajouter un message descriptif pour chaque commit afin de documenter les modifications apportées.











    Cloner un dépôt Git


    Une autre façon de démarrer un dépôt Git est de le cloner à partir d'un dépôt Git existant. Pour cela, on utilise la commande "git clone". Après avoir créé le dépôt sur GitHub, nous pouvons transmettre nos fichiers vers ce dépôt en utilisant les commandes suivantes :
    "git branch -M main" permet de renommer la branche principale en "main". Cette étape peut être nécessaire si la branche principale est nommée différemment.
    "git remote add origin [URL du dépôt GitHub]" ajoute le dépôt distant sur GitHub en tant que "remote" avec le nom "origin". Cela permet d'établir une connexion entre le dépôt local et le dépôt distant.
    "git push -u origin main" pousse les fichiers locaux vers le dépôt distant "origin" dans la branche "main". L'option "-u" configure la branche "main" du dépôt local pour suivre la branche "main" du dépôt distant.
    Après avoir effectué ces étapes, si nous vérifions le dépôt sur GitHub, nous devrions voir les deux fichiers transférés.

    Ajouter ou modifier des fichiers dans un projet et actualiser notre dépôt Git

    Pour enregistrer une modification ou l'ajout d'un fichier dans le dépôt Git, nous utilisons les commandes "git add" et "git commit", comme nous l'avons fait dans la leçon précédente. Il est important de noter que le commit d'un fichier est basé sur son état au moment de l'exécution de "git add". Ainsi, si vous modifiez un fichier après avoir exécuté "git add" et que vous effectuez ensuite un "git commit", seule la version du fichier au moment de "git add" sera enregistrée.
    Si vous souhaitez toujours enregistrer la dernière version d'un fichier, il est donc essentiel d'exécuter "git add" juste avant "git commit". Une autre option est d'utiliser la commande "git commit -a", qui ajoute automatiquement les fichiers modifiés et déjà suivis à la zone d'index avant de les valider, ce qui évite d'avoir à taper "git add" séparément.

    Consulter l’historique des modifications Git

    La manière la plus simple de consulter l’historique des modifications Git est d’utiliser la commande git log. Cette commande affiche la liste des commits réalisés du plus récent au plus ancien.

    Annuler des modifications apportées à un fichier

    Pour cela, nous pouvons utiliser la commande générale "git checkout -- nom-du-fichier" ou la nouvelle commande spécialisée "git restore". Ces commandes permettent de restaurer un fichier à son état précédent enregistré dans le dépôt Git.

    Qu’est ce qu’une branche ?

    Créer une nouvelle branche dans Git consiste simplement à créer un nouveau pointeur vers un commit spécifique, sans avoir besoin de recopier l'intégralité du répertoire de travail.
